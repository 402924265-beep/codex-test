# 洗碗机1月实际-同期差异核对网页小程序设计

## 目标

第一版做一个可核对模型，不直接替换“三张表”。小程序先跑在用户电脑上，同事在同一局域网通过网页地址访问，用来核对 `洗碗机1月` 的小科目差异。

核心目的：用户选择或输入财务报表路径后，小程序自动读取 25年同期和 26年实际数据，把 `26年实际 - 25年同期` 的差异显示到具体科目级别，并提供差异分析填写区。

## 第一版范围

包含：

- 只处理洗碗机项目的 `1月`。
- 自动读取或由用户输入以下源文件路径：
  - 25年同期实际：`E:\桌面\李想\财务报表\2025 monthly Renta DW _ NOV_ACT.xlsx`
  - 26年实际：例如 `E:\桌面\李想\财务报表\2026 MFG Variance Reporting_ DW_04.2026 v1.xlsx`，或当月实际报表中 `Renta DW _2026` 类 sheet
  - 三张表参考模板：`E:\桌面\李想\三张表\Candy洗碗机三张表-5.26.xlsx`
- 月度核对先不看预算，页面和导出里去掉预算相关字段。
- 输出科目级 `26实际 vs 25同期` 差异。
- 标出 25/26 科目不一致：
  - 两边都有
  - 25有26无
  - 26有25无
- 每个差异科目提供“差异分析”填写空。
- 显示总制造费到大科目的对比图。
- 支持导出 Excel 核对结果。

不包含：

- 不处理 `25年降费项目-洗碗机`、`26年降费项目-洗碗机`。
- 不生成最终“三张表”替换版。
- 不处理厨电。
- 不处理洗碗机2-5月。
- 不把预算作为差异判断口径。后续再加预算/4+8 对比。
- 不对成本数据 sheet 做第一版计算。后续扩展时，成本数据 sheet 的目标取 `4+8【最新】制造费用累计-大厨电（每月更新）-2026(3).xlsx`。

## 核对口径

科目级差异统一使用：

`差异 = 26年实际 - 25年同期`

字段：

- 科目编码
- 科目英文
- 科目中文
- 科目分类
- 25年同期制造费
- 25年同期单台
- 26年实际制造费
- 26年实际单台
- 单台差异 = `26年实际单台 - 25年同期单台`
- 额差异 = `26年实际制造费 - 25年同期制造费`
- 科目状态
- 差异分析
- 来源说明

预算字段不参与洗碗机1月页面输出，也不参与差异判断。

## 大科目口径

小程序需要按大科目汇总并画图，第一版至少支持这些组：

- 折旧
- 固定人工
- 固定费用
- 直接人工
- 变动费用
- 其他制造费

折旧的特殊口径：

`折旧 = Depreciation + Functional Currency Impact`

也就是图表和大科目汇总中，折旧不仅包含折旧科目本身，还要加上 FC/Functional Currency Impact（汇率影响）。科目级明细仍保留原始科目行，方便追溯。

大科目映射第一版做成配置，不能写死在代码里。后期要能方便加项目、改分类、增删大科目。

## 科目匹配规则

主键使用科目编码。

读取源表时：

- 从科目字段中提取 10 位数字编码。
- `FIX`、`VAR` 这种汇总行作为特殊编码保留，但页面默认和普通科目分开筛选。
- 同一编码如果出现多次，第一版按源表中该编码的当月制造费求和；单台用“汇总制造费 / 当月台数 * 1000”重算，并在来源说明中标记“同编码多行汇总”。

匹配输出：

- 两边都有：同时存在于 25年同期和 26年实际。
- 25有26无：25年有该科目，26年没有；26实际制造费和单台显示为空或 0，状态标明缺失。
- 26有25无：26年有该科目，25年没有；25同期制造费和单台显示为空或 0，状态标明新增。

例如 `6666021500 Repairs cost` 如果 25年有、26年没有，应在 `25有26无` 清单中清楚显示，不被静默丢掉。

## 可扩展性要求

第一版虽然只做洗碗机1月，但结构要方便后期加项目：

- 项目配置：洗碗机、厨电等项目通过配置新增。
- 月份配置：后续能扩展到 1-5月或全年。
- 源表适配器：不同财务报表格式用不同读取适配器，不把某一个文件格式写死。
- 科目大类映射：用配置维护科目到大科目的关系。
- 差异阈值：支持后续配置“单台差异超过多少”或“额差异超过多少”视为重点差异。
- 分析原因：用户填写的分析原因要能保存并导出，后续可复用到同科目。

## 页面设计

页面包含五个区域：

1. 文件输入和状态
   - 显示默认源文件路径。
   - 允许用户输入或选择 26年实际财务报表路径。
   - 显示月份：洗碗机1月。
   - 显示读取成功、缺文件、缺 sheet、缺列等状态。

2. 摘要指标
   - 25年同期科目数。
   - 26年实际科目数。
   - 两边都有科目数。
   - 25有26无科目数。
   - 26有25无科目数。
   - 总制造费差异。
   - 单台制造费差异。
   - 单台差异绝对值 Top 科目。
   - 额差异绝对值 Top 科目。

3. 大科目对比图
   - 显示 25同期 vs 26实际 的制造费对比。
   - 显示 25同期 vs 26实际 的单台对比。
   - 支持从总制造费钻到大科目，再从大科目查看明细科目。
   - 折旧图表值使用 `Depreciation + Functional Currency Impact`。

4. 科目级差异表
   - 支持排序：单台差异、额差异、科目编码。
   - 支持筛选：全部、两边都有、25有26无、26有25无、只看差异大。
   - 差异分析列可编辑。
   - 重点差异行用醒目但克制的底色标出。

5. 导出
   - 导出当前筛选结果。
   - 导出完整核对结果。
   - 导出未匹配科目清单。
   - 导出大科目汇总。

## 数据流

1. 启动网页服务。
2. 用户打开局域网网页。
3. 用户确认或输入源文件路径。
4. 后端读取 25年同期实际源表。
5. 后端读取 26年实际财务报表。
6. 后端按适配器提取 1月的制造费、单台、台数、科目编码、科目描述。
7. 按科目编码合并 25/26 数据。
8. 计算单台差异和额差异。
9. 按大科目映射汇总，折旧额外加 FC/Functional Currency Impact。
10. 返回给网页展示表格和图表。
11. 用户在网页中填写差异分析。
12. 用户导出 Excel。

## 脏 Excel 处理

实际财务报表可能存在 Excel 内部定义名异常。例如 `2026 MFG Variance Reporting_ DW_04.2026 v1.xlsx` 当前用 openpyxl 直接读取时会遇到 `#N/A is not a valid print titles definition`。

第一版实现必须考虑这类情况：

- 读取失败时，不直接让网页崩溃。
- 后端尝试用安全方式创建临时修复副本，例如清理无效 defined names 后再读取。
- 如果自动修复失败，网页显示清晰错误，提示用户另存一份 Excel 后重试。
- 原始源文件不被修改。

## 错误处理

如果发生以下问题，网页应直接显示可操作错误，不生成错误结果：

- 找不到源文件。
- 找不到目标 sheet。
- 识别不到月份列。
- 识别不到科目编码列。
- 某科目编码重复但无法汇总为数字。
- 报表格式不是当前适配器支持的格式。

如果只是某个科目 25/26 缺失，则不视为程序错误，而是作为核对结果显示在未匹配清单中。

## 技术方案

推荐使用本地 Web 小程序：

- 后端：Python + FastAPI
- Excel 读取：openpyxl，必要时增加临时修复/转换逻辑
- 图表：前端轻量图表库或原生 SVG/canvas
- 前端：简单 HTML/JavaScript 表格，第一版不引入复杂框架
- 运行方式：在用户电脑启动服务，监听局域网地址
- 输出：Excel 文件下载

这样第一版足够轻，便于快速验证口径。后续确认口径后，再扩展到更多月份、成本数据 sheet、最终三张表生成。

## 验证标准

第一版完成时，必须验证：

- 网页能在本机打开。
- 局域网地址可被同事访问，前提是 Windows 防火墙和网络允许。
- 用户输入 26年实际财务报表路径后，系统能读取或给出清晰错误。
- `洗碗机1月` 能成功读取 25年同期和 26年实际数据。
- `6666010188 Salary - Blue collar direct employees` 这类具体科目能显示：
  - 25同期制造费
  - 25同期单台
  - 26实际制造费
  - 26实际单台
  - 单台差异 = 26实际单台 - 25同期单台
  - 额差异 = 26实际制造费 - 25同期制造费
- `6666021500 Repairs cost` 这类 25/26 不一致科目能进入正确状态清单。
- 大科目图能显示总制造费拆分。
- 折旧大科目包含 Depreciation 和 Functional Currency Impact。
- 用户能填写差异分析并导出 Excel。
- 导出的 Excel 与网页当前数据一致。
